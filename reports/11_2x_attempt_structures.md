# 11. 「チャンピオン2倍」への挑戦 — バー内価格・建玉構造・位相・共通モードの徹底検証

日付: 2026-06-11 / 対象: `confluence_meanrev_v2` + 最適資金管理([[09]])/ 目標: **CAGR@DD20% の2倍(+43%超)**

**問い(ユーザー)**: レポートを分析し、より収益性の高いシステムを作れ。新しい仮説を立てて価格の動きを
徹底検証し、チャンピオンの2倍を目指せ。壁に当たったら見落としを再確認せよ。

**結論(先出し)**: 8つの新仮説(バー内実行アルファ×2 / M1粒度DD監査 / 分割決済 / TFアンサンブル /
分割エントリー / 位相オフセット / USDファクターゲート / 週末ギャップ・フェード)を全て実装・実測した。
**「同一テールリスク基準(bootstrap p95=20%)+約定現実性で比較すると、どの変更もチャンピオン単独
mp11 を上回らない」**ことが一貫して確認された。empirical 較正では +1.6〜+2.7pp 出る構成が複数ある
(分割エントリー統合 +26.5% 等)が、**全てブートストラップ理論テールの膨張(−29.4%→−32%)と
引き換え**=レポート10の ADX オーバーレイと同じレバ偽装の署名だった。唯一テールも改善した
週末ギャップ・フェード(robust ほぼ2倍)は、**エッジの全量が週初オープン15分以内に集中しており
約定現実性で棄却**。**2倍(+43%)は本データ・本ユニバースの正当手段では到達不能**が誠実な答えである。
副産物: (a) M1粒度DD監査で現行較正がほぼ誠実(誤差≈1pp, 推奨掛け目 k×0.95)と確認、
(b) 「即反転トレード(10%)が総利益の34%を運ぶ」という価格構造の発見、(c) ストリーム統合基盤
`ens_lab.py` と「empirical/robust 両較正+短保有は M1 監査」という検証プロトコルの確立。

---

## 0. 物差し(ヤードスティック)

- **基準**: champion v2 + 乖離連動z×max_pos([[09]])
  - empirical MtM DD=20%: mp8 **+21.62%** / mp11 **+23.83%**(本セッションで再現一致)
  - robust(bootstrap p95=20%): mp11 **+15.21%**
- **目標**: empirical +43% 超(2倍)
- 判定プロトコル: empirical と robust の**両較正**+IS較正→OOS素検証+高原性。
  empirical だけ上がりテール(p95)が悪化する構成は「レバレッジの別表現」であり前進と認めない
  ([[10]] の判例に従う)。

## 1. 仮説A: バー内(M1)実行アルファ — **棄却**(exp23)

全1214トレードの M1 パスを再構成(クロスは脚から合成)し、バー内の値動きを初めて実測した。

**価格構造の発見**(これ自体が本セッション最大の知見):
- 勝ちトレードでも 58% は1バー以内に 0.5σ 以上の逆行を経験する(4バー以内なら 76%)。
- 負けトレードの逆行は深い(4バー以内中央 2.17σ vs 勝ち 1.14σ)— ただし分布の重なりが大きく
  個別予測には使えない([[08]] の AUC≈0.56 と整合)。
- **一度も 0.25σ すら押さない「即反転」トレード(全体の10%)が総利益の34%を運ぶ。**

これが実行改善を全て封じる:
- **指値エントリー置換**(entry−δσ に TTL バー指値、未約定見送り): δ×TTL 20通り全てで総リターン減。
  約定分は +4〜24bps/トレード改善するが、即反転の最良トレードを取り逃がす機会損失が常に上回る。
- **指値決済**(z=−0.5 相当価格を前バー統計で因果計算しバー内タッチで約定): タッチ率99%でも
  平均 **−5.1bps**。終値確認の出口が支配する(出口最適性の確認は[[04]][[08]]に続き3度目)。

→ **チャンピオンの終値確認シグナルはバー内構造を既に最適に刈り取っている。実行レイヤーに残された
アルファは無い**(むしろ現行が指値系より優れる)。

## 2. 仮説B: M1粒度 MtM DD 監査 — **較正はほぼ誠実**(exp24)

H4終値グリッドの DD=20% は、M1終値グリッドで測り直すと **−21.0%**(過小評価 ≈1pp)。
M1粒度で 20% に再較正すると k=7.98→7.60、CAGR +21.62%→**+20.52%**。
→ 既存プロトコルの結論を覆さない誤差。最終運用時は k に ×0.95 の掛け目を推奨。

## 3. 仮説C: 分割決済(scale-out)— **棄却**(exp26、鉄の三角形・未踏領域②を閉鎖)

2レグ出口(早出し w を exit_z=a で、残りを b で決済)を 6組合せ×3ウェイト、
専用シミュレータで統合 CAGR@DD20 比較。**全18構成がベースライン未満**
(最良 a=1.0,b=0.5,w=0.25 で +20.33% < +21.62%)。早出しで k は 8.6〜9.1 に上がるが
取り損ないが常に上回る。深掘り方向(b=0.0)も劣後。**出口は touching 禁止が最終確定**。

## 4. 仮説D: 時間足アンサンブル — **棄却**(exp25/27)

- **H1スケール**(同一構造・同一バー数): プールPF **1.02**・IS赤字(0.95)・最悪年 −0.42。
  コスト後エッジ消滅([[02]] の「H4が最適」を再確認)。entry_z を絞っても悪化。
- **D1スケール**: 単体は本物(プールPF1.55, IS1.46/OOS1.66, **H4との月次相関 −0.05**)。
  しかし**DD効率が悪い**(保有22日の含み損エクスカーションが大きく、D1単独 CAGR@DD20 = +4.7%)。
  統合の最良 +22.16% < mp11 +23.83%。「単体黒字・無相関」でも、リターン/含み損 効率が劣る
  ストリームを混ぜるとDD予算の使い方として損([[10]] の構図の時間軸版)。

## 5. 仮説E: 分割エントリー(scale-in)— **同一テールで純増ゼロ**(exp28、未踏領域③を閉鎖)

深押しストリーム(entry_z=2.5、他は完全固定)は単体で**チャンピオン超えの質**
(プールPF1.91, OOS2.15, 最悪年≈0)。64%はチャンピオン保有中に発生=事前計画2トランシェ建玉と等価。

- empirical 較正: champ:9+deep:5(w=0.5)で **+26.52%**(基準+23.83%、OOS +32.85%、毎年プラス、
  w=0.5 の5構成全てが基準近傍以上=高原)→ 一見ブレイクスルー。
- **robust(p95=20%)較正: +15.21% vs 基準 +15.21% — 完全に同着。** p95 は −29.4%→−32.0% に膨張。

→ **empirical の +2.7pp は「観測1パスでテールが顕在化しなかったレバレッジ」**。
[[10]] の ADX overlay と同じ罠。真の前進ではない。
(なお計画的トランシェはマーチンゲール/ナンピン(損失追撃)非該当だが、採否以前に効果がない。)

## 6. 仮説F: 位相オフセット(H4グリッドを+1/+2/+3hずらした同品質ストリーム)— **棄却**(exp29)

ストリーム品質は均一(PF 1.61〜1.74、OOS PF 全て1.68+、パラメータ再最適化なし)、相互相関 0.64〜0.80。
4ストリーム統合 mp11×4: empirical **+25.43%**(+1.6pp)/ Sharpe 1.29 — だが p95 −31.9% に膨張、
**robust 較正では +14.47% < 基準 +15.21%**。運用複雑性4倍で同一テール価値はマイナス。棄却。

## 7. 仮説G: USDファクター・トレンドゲート(共通モード狙撃)— **方向は正しいがノイズ範囲内**(exp30)

ここまでの連敗の構造的理由: **テール episodes(2022型 USD主導トレンド)が族内の全ストリームに共通**
で、何を足しても同じテールへのエクスポージャが線形に増えるだけ。既存フィルタは全て銘柄単位
(ER40/ボラ/slow_z)だったので、最後の見落とし候補として**ポートフォリオ共通モード=USDファクター
自身のトレンド強度(7メジャー等加重 log 因子の Kaufman ER(40))**でエントリーを絞るゲートを検証した。

- 全銘柄一律ゲート: ほぼ全構成で劣化または中立。
- **USD脚ペアのみゲート(th=0.30〜0.35, g=0〜0.5)**: 本セッション唯一、**robust(同一テール)較正で
  基準を上回った**(n_boot=600 で +15.9〜16.1% vs +15.2%)。4構成全てプラス=高原あり、
  ストーリーも因果的(ドル因子が一直線の時に USD ペアの逆張りを止める)。
- **ただし高精度検証(n_boot=1500, 複数シード)で優位は +0.2〜0.6pp に縮小**。p95 推定のシード間
  ばらつき(±0.4〜0.8%abs)から来る較正ノイズと同程度であり、OOS empirical はわずかに基準を下回る
  構成もある。**統計的に分離できないため採用見送り**(方向性の記録として残す)。

## 7.5 仮説H: 週末ギャップ・フェード(新規エッジ族)— **データ上は実在、約定現実性で棄却**(exp31)

週末ギャップ(金曜終値→週初オープン)のフェードは、コスト後・保有4hで +6.6bps/トレード
(10pip以上のギャップ, 年126件, IS+3.5/OOS+9.8)の実エッジに見え、統合すると本セッション唯一の
「empirical↑かつ p95 改善」(robust +15.4→+30.1%=ほぼ2倍!)を示した。**だが敵対監査で崩壊**:

- エントリーを週初オープンから **15分遅らせるだけで +6.6→+0.4bps(IS は −2.7bps に転落)**。
  30分遅延で −1.0bps。**エッジの全てが「週初の最初の15分」**=スプレッドが数倍に開き、
  リテールの約定が最も非現実的な瞬間に集中している。
- スプレッド感応度(×2→×4)では生き残る(+4.6bps)ため、問題はコストでなく**「オープン価格で
  即時に約定できる」という仮定そのもの**。
- もう一つの罠も特定: 保有4h≈H4 1バーのストリームは **H4終値グリッドの DD にリスクがほぼ写らない**
  (マークの間に建玉が現れて消える)ため、較正が k を過大評価する。短保有ストリームの統合は
  M1粒度 DD 監査が必須(プロトコルとして記録)。

→ 棄却。ただし「ギャップは最初の15分で埋まる」という価格構造の知見と、短保有ストリームの
監査プロトコルは資産。

## 8. メタ結論

1. **CAGR@同一テールDD のフロンティアは「チャンピオン単独+乖離連動z+mp11」で実質的に固定**。
   実行(バー内)・出口分割・エントリー分割・時間足・位相 — 自由度を7方向に動かしても、
   同一テール基準で +0.1pp を超える前進は1つもなかった。
2. **empirical 較正だけを物差しにすると +2〜3pp の「見かけの改善」が量産できる**(本セッションだけで
   3構成)。全て p95 膨張つき=レバの別表現。この署名(empirical↑ & p95↓)は過剰最適化検出器として
   有効で、今後の検証でも必須チェックとすべき。
3. **2倍(+43%)の正直な評価: 本データ(7メジャー+合成クロス, 2016-2026)・カーブフィット禁止の
   制約下では到達不能**。フロンティアを動かす残り筋は (a) 別資産クラス/別データソースの追加
   (テール episodes が真に異なるエッジ)、(b) 実弾前向き運用での検証続行、のみ([[10]] と同結論)。
