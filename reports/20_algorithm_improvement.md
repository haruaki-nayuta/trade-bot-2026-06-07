# 20. アルゴリズム改善ラウンド(FX限定・リスク契約固定) — 3層全て閉鎖 / rolling z は欠陥ではなく機能

日付: 2026-06-12 / 対象: `confluence_meanrev_v2_d1` + P4.0 mp8([[18]][[19]]) /
制約: **リスク契約固定(robust p95 DD=20%)・金/銀/原油/株価指数の追加禁止・既存パラメータの再調整禁止**

**問い(ユーザー)**: 手法のアルゴリズム改善(構造の追加・置換)で収益をもっと増やせないか。

**結論(先出し)**: **増やせない。本ラウンドの採用もゼロ**。reports/19 までで閉鎖済みの層
(閾値・出口・配分形状・用量・実行タイミング・mp・位相・USDファクター)を除いて残っていた
未踏のアルゴリズム層は3つ — ①z推定器の置換 ②WF-ML エントリー拒否 ③WF-ML ランクサイジング —
で、3つとも事前登録プロトコルで明確に死亡した。副産物として
**「rolling z の窓挙動はエッジの構成要素」「ML 序列は実在するがテール中立」**という
2つの構造的知見を確定した。同一リスク契約での天井は robust +18.6%(実効 +17.7%)のまま。

---

## 1. exp59 — z 推定器の置換: 全滅(-13〜-18pp 級)。rolling z は load-bearing

**仮説**: 10年間固定の rolling mean/std(50) は「ボライベントが窓内にある間 z が人工的に縮む/
窓落ちで跳ぶ」構造欠陥を持ち、EWMA(滑らかな忘却)や median-MAD(外れ値頑健)に置換すれば
同じ閾値のままで乖離測定の品質が上がるはず。

**設計**: 推定器のみ差し替え(EWMA span=窓、MAD 定数1.4826 の標準形固定=内部チューニング禁止)。
entry/exit/slow/d1ゲート/サイジング z_entry に一貫適用。6変種固定。base はプール完全一致を検算。

**結果(seed0)**: 全変種が論外の大差で死亡。
| 変種 | n | プールPF | 口座 rob_s0 |
|---|---|---|---|
| base(roll) | 1207 | 1.77 | **+18.18%** |
| ewma_short | 402 | 1.86 | +4.55% |
| ewma_both | 166 | 2.03 | +3.56% |
| mad_short | 486 | 1.47 | +3.56% |
| mad_both | 153 | 1.11 | +0.25% |
| and_conf(roll∧ewma) | 133 | 2.08 | +4.77% |

**機構(EURUSD H4 実測)**: P(|z|>2) = roll 12.7% / **ewma 2.2%** / mad 8.1%。
- EWMA はボラスパイクを指数的にしか忘れない→分母が恒常的に大きい→閾値イベントが 1/6 に崩壊。
  プール品質は微改善(PF1.86)しても、DD予算口座は取引頻度が落ちると k で取り返せない
  (鉄の三角形のアルゴリズム版)。
- MAD は直近ボラの「大きさ」を無視する(中央値分母は嵐で育たない)→ rolling-std が
  自動回避していた嵐の中に入る → 品質自体が劣化(PF1.47・勝率64%・プール負け年3)。
- 確認型合流(and_conf)は PF2.08 と最高品質だが n=133=年12取引で口座を駆動できない。

**確定知見**: rolling std の「窓汚染」は欠陥ではなく機能 —
**(a) イベント直後は分母膨張で自動的にエントリー抑制(嵐の中の逆張りを防ぐ)、
(b) 窓落ちで完全忘却して再武装する**。閾値群(entry_z=2.0 等)はこの推定器のスケール・
時間特性と共進化しており、推定器だけの置換は不能。**z推定器の軸は閉鎖**
(再開には閾値の同時再調整が必要=カーブフィット禁止域)。

## 2. exp60 — WF-ML エントリー拒否: 拒否コホートが OOS で純プラス=原理的に不成立

**仮説**: reports/08 は単一特徴量フィルタを網羅し ER のみ生存。特徴量の**相互作用**
(深z×高ボラ×持続トレンド等)は非線形モデルでしか拾えず未踏。

**設計(事前登録)**: 因果特徴量15列(シグナルバー確定値のみ: z深さ/slow_z整合/ER/RSI距離/
ボラ分位/ATR%/直近リターン×方向/傾き×方向/高安距離/時刻/曜日/クロス・JPYフラグ)。
完全因果WF(年初までに **exit 確定**したトレードのみで学習→当年に適用、2019開始・burn-in 3.5年)。
モデル2種(HistGBR/Ridge)×拒否用量2種(学習分布の下位10/20%)固定。

**結果**: 4構成全てで**拒否コホートの実現リターンが純プラス**
(mean +2.5〜+7.0bps、勝率60-67%、sum +0.02〜+0.12)= プール段ゲートで全滅。
口座に乗せる以前に「外すべき負けトレード」を WF 外挿では特定できない。
reports/08/15 の「塩漬けはエントリー時点で予測不能」が**相互作用込みでも**成立と確定。

## 3. exp61 — WF-ML ランクサイジング傾斜: 序列は本物だがテール中立=口座利得ゼロ

**着眼(exp60の副産物)**: 拒否は不可でも、採用側 +17.8〜19.4bps vs 拒否側 +2.5〜7.0bps の
**序列(ランキング能力)は実在**する。トレードは全部取り、配分だけ WF 予測で傾ける
(乖離連動 f(z) の直交補完。z は特徴量に含まれるため限界価値=z以外の相互作用)。

**設計(事前登録)**: 形状2種固定(half=中央値で±25% / quint=五分位 0.6〜1.4倍)×モデル2種。
`alloc = equity*(k/mp)*(f(z)/fbar)*(m/mbar)`(k線形性維持)。burn-in は m=1。

**結果**: プール段の序列は明確(ridge_quint: m>1 側 +30.4bps vs m<1 側 +8.3bps)なのに、
口座 5シード平均は **gbr_quint -0.39pp / ridge_half +0.02pp / ridge_quint -0.21pp**
= 全構成ノイズ以下、全シード正なし。機構: 高予測トレードへの傾斜は **MtM テールを薄くしない**
(emp_k が 8.89→7.55-8.03 に低下して帳消し)。reports/16 の g(vol) サイジング・reports/09 の
ケリーと同じ死に方=**「平均リターンの序列」と「DD予算下の配分価値」は別物**。
f(z) 乖離連動だけが両方を満たす例外であり続けている。

## 4. 総括 — チャンピオンの改善余地は全層で尽きた

reports/14 以降の累計で閉鎖された軸: 閾値・出口・浅z配分(15) / vol_sd・g(vol)・25銘柄・
mp11・指値(16) / 実行タイミング d≥2・位相(18) / mp9-10・h20・P・τ・Δz・z鮮度・出口遅延・
週末衛生(19) / **z推定器・ML拒否・MLランク傾斜(本ラウンド)**。
ER フィルタ(08)・P=4.0(14)・d1(18) の3つだけが10年データで正当に採用できた改善であり、
**同一リスク契約(robust p95 DD=20%)の天井は +18.6%(M1実効 +17.7%)** が最終結論。
これを動かせる残りのダイヤルはリスク予算([[19]]: p95=22%で+20.4%)か、
追加が禁止されている新資産クラスのみ。
