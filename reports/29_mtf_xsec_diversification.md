# 29. 収益+10%第3ラウンド(金以外) — マルチ時間足/xsec/統計裁定/トレンド/Hurst の確定棄却(web手法含む)

日付: 2026-06-13 / 対象: `confluence_meanrev_v2_d1` + z-power P4.0 mp8([[18]][[19]]) / 制約: **金を含む銘柄は使わない**

**問い(ユーザー)**: チャンピオン手法の収益をさらに 10% 以上伸ばせ(金以外)。改善策をリサーチし適応可能性を徹底検証せよ。

**結論(先出し)**:
- **採用ゼロ**。サーベイで「唯一未検証」と判明した**マルチ時間足アンサンブル**・未統合だった**xsec**に加え、
  ユーザー要望で**web/文献の標準手法**(Avellaneda-Lee 統計裁定・Hurst/分散比フィルタ・trend+MR オーバーレイ・メタラベリング)も
  robust 較正口座で確定検証 → **全て棄却**(§4.5)。
- 機構は同一:**第2のブックが単独で弱すぎ(Sharpe ≪ 1.37)、ゼロ相関でも DD予算を食うだけで、失血窓では稼がない(むしろ沈む)**。
  チャンピオンを分散できる強いFXエッジは存在しない=本プロジェクトの核心結論([[10]] 「メジャーFX補完エッジは単体で弱すぎる」)を**分散の側からも確証**。
  唯一の例外だった金 tsmom([[gold-tsmom-first-true-advance]])は別資産クラスで、今回は禁止。
- 同一リスク契約での天井 **robust +18.2%(=[[19]] の +18.63%, 本ラウンド3シード測定 +18.18%)を再確認**。
  **+10% の唯一の正直な道はリスク予算の変更のみ**: robust p95 目標を 20.0%→**約21.7%** にすると CAGR **+20.0%**(実測・[[19]] の表と 0.06pp 一致)。
  これは手法改善ではなく**リスク選好の選択**=ユーザー判断事項([[19]] の線引きを踏襲し、本ラウンドでも無断採用しない)。

  これは [[18]][[19]] の確定値 +18.63%(5シード)とシード集合差の範囲内で一致。
- サーベイ(全28レポート+research/横断)で「改善レバー」の既検証/未検証を棚卸し → **未検証はマルチ時間足アンサンブル(=選択でなく統合)1本のみ**。
  スワップ実測は外部データ待ち([[10]][[14]])、相関連動配分([[14]] 通貨ネッティング)・リスクパリティ([[04]])・各種オーバーレイ([[10]])・サイジング鮮度/mp/P/出口遅延([[19]][[20]])は全て検証済み閉鎖。

## 1. マルチ時間足アンサンブル(exp59/60)— 弱ブックは独立サイジングしても DD予算を食うだけ

**仮説**: チャンピオン(平均回帰)を H4 だけでなく D1(同一バーパラメータ=多週間の遅い平均回帰)でも回し、両ブックを1つの DD較正口座に統合すれば、
時間軸の分散で「同一 p95=20% でより高い k」が引けて CAGR が上がるのでは。reports/10 の弱いトレンドオーバーレイ(保険)とは別物=実証済みの正のエッジの加法的分散。

**単独成績(exp59, robust p95=20%, 3シード)**:
| 構成 | robust CAGR | Sharpe | プラス年率 | 取引/年 |
|---|---|---|---|---|
| H4(ベース) | **+18.18%** | 1.37 | 100% | 110 |
| D1 | **+1.34%** | **0.25** | 64% | 21 |
| H1 | **−0.38%** | −0.10 | 45% | 423 |

→ D1 は「実在するが極めて弱い」エッジ、H1 は**死亡**(コスト支配=[[12]] の短期足結論と整合)。
グリッド近似の健全性チェック合格(D1 pool を H4 グリッドで MtM 評価 → +1.25% ≈ D1グリッドの +1.34%)。

**月次相関 corr(H4, D1) = −0.025 ≈ ゼロ**(分散の素地は理想的)。にもかかわらず:

**素朴統合(共通 k・共通 max_pos)は +8.9% に半減**。原因=D1 の高い1トレード MtM ボラが口座全体の k を 6.08→2.83 に引き下げ、優秀な H4 を兵糧攻め。

**欠陥を正した「正しい分散テスト」(exp60: 独立サイジング+独立スロット H4=8/D1=8, D1 を weight α で縮小, robust p95=20% に較正)**:
| α(D1相対倍率) | CAGR | p95 | empDD | worstY |
|---|---|---|---|---|
| 0.00(純H4) | **+18.20%** | -19.3% | -14.0% | +3.7% |
| 0.10 | +17.79% | -19.7% | -13.4% | +4.5% |
| 0.20 | +17.02% | -19.9% | -12.6% | +5.1% |
| 0.50 | +13.86% | -20.0% | -12.0% | +2.7% |
| 1.00 | +9.43% | -20.0% | -10.9% | +0.4% |

→ **最良 α=0.00**。D1 を少しでも混ぜると CAGR は**単調に減少**(=内部最適点なし=カーブフィットの余地すらない、クリーンな棄却)。

**失血窓テスト(決定的)**: H4 の最悪十分位月(n=12)に D1 は平均 **−0.013%**(≒ゼロ、プラスでない)。
→ D1 は H4 のテール(失血窓)を**全く守らない**。だから「ゼロ相関による Sharpe算術の天井(√(1.37²+0.25²)=1.39 ≒ +1.5%)」すら口座段では実現せず、DD制約下では純減になる。

## 2. クロスセクション平均回帰 xsec(exp61)— 同じ死に方+失血窓で逆に沈む

xsec_meanrev = 19銘柄を vol正規化リターンで順位付け→最も負けロング/最も勝ちショートのポートフォリオ・ランキング戦略([[02]] 系)。
機構がチャンピオン(per-instrument)と異なり月次相関 0.135、docstring 上「真価は分散」。ただし単独 PF1.10・常時8脚で資本を食う弱エッジ。

**DD較正統合(exp61: H4=z-power本番 / XSEC=flat, 独立スロット8/8, weight α スイープ, robust p95=20%)**:
| α(XSEC倍率) | CAGR | p95 | empDD |
|---|---|---|---|
| 0.00 | +18.25% | -19.3% | -14.0% |
| **0.10** | **+18.70%** | -20.0% | -13.3% |
| 0.20 | +16.59% | -20.0% | -15.1% |
| 0.50 | +10.56% | -19.3% | -15.8% |
| 1.00 | +6.55% | -19.2% | -14.2% |

→ 最良 α=0.10 で **+0.45pp(+2.4%相対)**。**較正ノイズ帯(±0.4-0.8pp/シード)の内側**=実質ゼロ([[22]] のピラミッド +0.34pp と同じく採用バー未満)。
**失血窓テスト**: corr(H4,XSEC)=+0.026、H4 最悪十分位月で XSEC 平均 **−0.243%**(=チャンピオンと一緒に沈む。テール保護どころか逆相関の保険にもならない)。

## 3. リスク予算ダイヤルの精密化(exp62)— 唯一の正直な +10%

P=4.0 d1 mp8 で robust 較正目標 p95 DD を刻んだ実測(3シード):
| p95 目標 | robust CAGR | k | 経験的(単一パス)最大DD | 理論DD p99 |
|---|---|---|---|---|
| **20.0%(現行契約)** | **+18.18%** | 6.08 | -14.0% | -23.5% |
| 20.5% | +18.71% | 6.25 | -14.4% | -24.1% |
| 21.0% | +19.23% | 6.41 | -14.7% | -24.7% |
| **≈21.7%** | **+20.00%(+10%相対)** | ≈6.65 | ≈-15.2% | ≈-25.5% |
| 22.0% | +20.30% | 6.74 | -15.4% | -25.8% |
| 23.0% | +21.39% | 7.08 | -16.2% | -27.0% |

→ **+10%相対(+20.0%)に必要なのは p95 を 20.0%→約21.7% に緩めること**(=「20回に1回級の理論DD」を 1.7pp 深く許容)。
経験的最大DD は -14.0%→約-15.2% と僅増にとどまる。**[[19]] の表(22%→+20.36%)と本実測(22%→+20.30%)が 0.06pp 一致**=機械の妥当性確認。
なお本番既定の **empirical 較正なら現行のまま +27.50%**(ただしその時の理論テール p95 は -27.3%)=「どの DD 解釈で運用するか」の問題でもある。

## 4. 機構:なぜ FX 分散は効かないのか(二重苦)

DD≤20% 口座に第2ブックを足して CAGR を上げるには、そのブックが
(a) **単独で十分に高い Sharpe**(チャンピオン 1.37 に対抗できる)を持つ、か
(b) **チャンピオンの失血窓で稼ぐ**(テールを削って k を上げる)、のどちらかが要る。
メジャーFXの第2エッジ(D1平均回帰 0.25 / xsec ~0.3 / トレンド系=net負[[10]])は**両方とも満たさない**:
- (a) を満たすほど強いFXエッジはチャンピオン以外に存在しない(本プロジェクトの核心)。
- (b) も外れる:D1 は失血窓でゼロ、xsec は失血窓でマイナス(平均回帰系は持続トレンドで一緒に沈む)。
ゆえにゼロ相関でも「弱ブック×DD制約」で純減。**唯一 (a)(b) を満たしたのが金 tsmom**(別資産クラス・トレンド)で、今回は禁止。
=[[05]] 鉄の三角形/[[10]] の再々確認。チャンピオンは自分のリスク予算を既に効率使用しており、ボルトオン分散ではフロンティアを外へ動かせない。

## 4.5 web リサーチで推奨される手法の検証(ユーザー要望「web に転がってる改善策も試せ」)

「リスク予算が唯一の道」への差し戻しを受け、web/文献の標準的改善手法を**実際にチャンピオンへ適用して検証**した(WebSearch: 統計裁定/レジーム検出/ポートフォリオ構成)。**全て棄却**。

| web手法 | 出典 | 検証(exp) | 結果 |
|---|---|---|---|
| **ドル中立 残差平均回帰(Avellaneda-Lee型 統計裁定)** | stat-arb 文献 | exp64 | 棄却。共通ドルファクター除去で「失血窓を守る」仮説を立てたが**反証**: 単独 Sharpe 0.54(≪1.37)・**相関+0.346(正)**・**失血窓 −1.24%(一緒に沈む)**。beta較正版はさらに悪化。xsec の精密版だが同じ死。 |
| **Hurst指数 / 分散比(VRT)エントリーフィルタ** | Hurst文献 | exp63 | 棄却。チャンピオンプール(ER≤0.55済)で **AUC 0.508=無情報**・除去コホートは**純プラス**(儲かるトレードを捨てるだけ)・ER層別後は逆符号。**ER の冗長な焼き直し**(reports/08 で ER が勝った理由の再確認)。 |
| **トレンドフォロー・オーバーレイ(trend+MR)** | 2024-26 practitioner 多数 | exp65 | 棄却。tsmom(H4/D1)・breakout の4書とも**失血窓ではプラス(+0.30〜+0.68%)=正しい凸性を確認**したが、**単独が net 負(CAGR −2.7〜−4.1%, Sharpe −0.6〜−0.95)**。統合は weight に対し CAGR 単調破壊(α0.25 で +9% へ半減)。**保険料(平時ドラッグ)が失血窓便益を圧倒**=reports/10 を現チャンピオンで再確認。 |
| **メタラベリング(López de Prado)** | LdP 2017 | reports/20 で検証済 | 棄却(再検証不要)。2次モデルで take/pass+サイズ決定=reports/20 の WF-ML 拒否(拒否コホートOOS純プラス=負け予測不能)+WF-ML ランクサイジング(テール中立)と同一。QuantConnect も "not a silver bullet"。 |

| レジームフィルタ(トレンド回避) | MR 実務記事 | 採用済 | ER≤0.55 + slow_z + vol-calm が既に実装。 |

**構造的結論(web手法がFXで効かない理由)**: FX の3大ファクターは MR(チャンピオン=強い)・トレンド(メジャーでは net 負=ドリフトベータ[[17]])・キャリー(シグナル化はリーク[[14]]/実勢スワップは外部データ)。
チャンピオンは**唯一プレミアムのある MR を既に harvest** 済み。web の「trend+MR は無相関で相補」は**両脚が正の株/先物では真**だが、**FX メジャーはトレンド脚が net 負**なので統合は drag に負ける(exp65 が実証)。MR を MR で分散はできない(同じ持続トレンドテールを共有, exp60/61/64)。唯一(a)正の単独リターン (b)失血窓で稼ぐ を両立したのが金 tsmom([[gold-tsmom-first-true-advance]])=別資産で禁止。

## 5. プロトコルへの追加(恒久)

1. **分散レバーの事前スクリーニング2点**: 統合 DD較正(重い)に入る前に、候補ブックの (i) **単独 robust Sharpe** と
   (ii) **チャンピオン失血窓(最悪十分位月)での条件付き平均リターン** を測る。両方弱ければ統合は必ず純減=計算を省ける
   (D1=Sharpe0.25/失血窓0%、xsec=失血窓-0.24% で予言どおり死亡)。
2. **マルチ時間足は「選択」と「アンサンブル」を区別**: H4 が最良の単一足という選択結論([[01]][[02]])と、複数足の同時運用(本ラウンドで棄却)は別問題。後者は閉鎖。
3. **「ゼロ相関=分散の素地」は必要条件であって十分条件でない**: corr≈0 でも単独 Sharpe が低ければ DD制約下で純減(D1 の判例)。相関だけ見て混ぜない。
