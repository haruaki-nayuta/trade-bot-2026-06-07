# 30. 通貨限定の相補手法でポートフォリオ底上げ — 独立スリーブ・レンズ + レバ偽装監査

日付: 2026-06-13
対象: `confluence_meanrev_v2`(H4・19銘柄・無ストップ平均回帰, z-power P4 / mp8 / 20%DD較正)
ユーザー goal: チャンピオンを補う**通貨限定(金・非通貨禁止)**の相補手法を追加し、足を引っ張らずに利益を底上げ(ポートフォリオ化)。追加手法は「相応にロバスト & 確実に利益」が要件。

> ⚠️ **後続の重要な訂正(reports/31)**: 本報告の「通貨内に底上げ補完なし」は **長い足(D1中心)の探索**での結論。
> ユーザー発案で**時間足を H1 に下げる**と話が変わり、**USDJPY H1 の ~1日モメンタム**が全敵対検証を生き残る相補スリーブとして発見された
> (robust CAGR +0.9〜2.3pp・レバ偽装なし・負相関・失血窓で稼ぐ)。下記の結論は「D1・per-instrument/xsec」に限定して読むこと。

## 結論(先出し)

**長い足(D1)・per-instrument/xsec の通貨限定補完では「利益を底上げするロバスト手法」は存在しない**を、reports/10/29(共有DD)とは別の
**独立スリーブ・レンズ**で独立に再確認した(短い足 H1 の例外は reports/31)。新しい知見は2つ:

1. **「単体ロバスト黒字」と「失血窓で稼ぐ」は構造的に反相関**。チャンピオンの失血窓(=中庸ボラ×持続USDトレンド)を
   守る候補(トレンド追随)は全て**単体赤字**(平時whipsawの保険料)。単体黒字で直交する唯一の候補(xsec)は
   **失血窓を守らない**(IS無保護)。ゲートで平時ドラッグを除くと**失血窓保護も同時に消える**(早期トレンド捕捉が保護の源)。
2. **xsec を独立スリーブで足すと「足は引っ張らない」が「底上げもしない」**。
   経験的DD較正では w=0.15 で +1.75pp に見えるが、**bootstrap p95 が 0.285→0.298 に悪化=レバ偽装**。
   robust(p95=20%)較正の正直な純増は **+0.28pp(≈ノイズ)**。xsec は安全な分散材だが利益ブースタでない。

→ チャンピオンは近Pareto最適([[05]][[10]][[champion-profit-anatomy]])。通貨内に底上げレバーは無い。
正直な利益レバーはリスク予算のみ(p95 20→約21.7%で+20%、ユーザー判断)、または金解禁([[gold-tsmom-first-true-advance]])。

## 1. 失血窓の再特定(補完の必要条件)

`bleed_lab.py`: チャンピオンMtM equity が水面下>-5% の月 = **18/121ヶ月(15%)**。
- 失血窓は **ER高め(0.184 vs 平時0.172, 比1.07)・ボラ中庸(やや低 0.0019 vs 0.0020)**。
- **2021-22に集中(12/18ヶ月)**。最深=2022-03〜06のUSDラリー(DD -15.5%, ER 0.19-0.22)。
- = 利益の正体(reports/15)と整合: 怖いのは嵐でなく「だらだら続く持続トレンド」。

→ 理想の補完 = **中庸ボラ×持続USDトレンド(高ER)で稼ぐトレンド追随**かつ**単体ロバスト黒字**。

## 3. ゲートで両立できるか → 不能(構造的, `exp_usdbasket_gated` / `exp_basket_scan`)

USDバスケットTSMOM(共通ドルファクター集約=単ペアのidiosyncraticノイズ相殺)に**平時フラット化ゲート**を試す:
- **ERゲート(er≥0.2)で平時ドラッグ除去 → total -4364→+2238 に改善するが、失血窓保護が反転**(bleedIS +123→-142)。
  機構: 高ERゲート=トレンドが成熟してから入る → チャンピオンが轢かれる**初動**を取れず、成熟期の反転でwhipsaw。
  **早期トレンド捕捉(=失血窓保護の源)と平時whipsap回避はトレードオフ**で、高原は存在しない。
- 単一lookbackスキャン(D1 21〜252, H4 120〜1080): **単体プラスは D1 L=63/84 のみ**で**IS負の単一セルfluke**。
  pos_yr 最大55%、worst_yr -6000超。**「FXトレンド全滅」はバスケットレベルでも成立**(2016-26は低トレンドFX期)。

## 4. xsec 独立スリーブの厳密評価 + レバ偽装監査(`exp_two_sleeve`)

各スリーブを**単独で20%MtM DDに較正**→重みwで合成→**20%DDにレバ調整したCAGR**を比較。
reports/29(記憶)の共有DD口座(スロット競合で純減)と違い、独立スリーブは月次相関0.043の分散が効く。

- **見かけの+1.75ppは大半がレバ偽装**: 経験的最大DDは下がる(-20→-16.6%)が bootstrap p95 は**悪化**(0.285→0.298)。
  xsec の太い尾(クロスセクションMRの急逆行)を、経験的に浅く見えるDDを根拠にレバ上げで露出させている。

## 4.6. 「トレンド赤字は実装か?エッジ不在か?」の決着(Workflow 8エージェント, ユーザー発案)

ユーザーの問い: 「トレンドは勝率を下げて1トレードで大きく取るもの。実装(特にエグジットが勝ちを切る)が悪いのでは?」
→ gross/net分解 + CTA標準のbest-practice順張りを5変種再構築して切り分け(`exp_trendA-E`, `diag1/3`, `exp_regime_gated_trend`)。

**決着: 実装は原因でない=エッジ不在(資産のトレンド性不足)が原因。**

| 実装 | GROSS Sharpe | NET Sharpe | net黒字ペア | 備考 |
|---|---|---|---|---|
| naive tsmom / breakout / adx | -0.09 / -0.20 / -0.03 | -0.11 / -0.21 / -0.04 | EURUSD等のみ | D1低回転でgross≈net |
| A: vol-targetアンサンブル(CTA標準) | -0.052 | -0.104 | 2/7 | plateau=頑健に負 |
| B: リスク調整モメンタム+ヒステリシス | -0.229 | -0.232 | 2/7 | OOSさらに悪化 |
| C: バッファ付きブレイク+利伸ばしトレ | -0.272 | -0.278 | 1/7 | 工夫がむしろ悪化 |
| D: 分散バスケット(USD共通ファクター) | -0.063 | -0.122 | 0 | 分散でゼロに寄るが越えない |
| E: 低回転 W1/月次(コストほぼ0) | -0.085 | -0.087 | 4/7 | 回転0でもgross負 |
| (対照)金tsmom 同手法 | **+0.596** | **+0.596** | — | ER0.181>メジャー0.158 |

3つの決定的事実:
1. **GROSS(コスト0)でも全実装が負**。vol-target/アンサンブル/no-tradeバンド/ヒステリシス/確認バッファ/利伸ばし/分散/低回転を全部試しても符号は1つも正に変わらず(全plateau=頑健な負)。
   コストドラッグは実在(H4でD1の約5倍)だがD1ではgross段階で負=「コストが殺す」でも「実装が悪い」でもない。
2. **損小利大は保たれている**(エグジットは勝ちを切っていない): payoff比 tsmom 1.9-2.1 / breakout 1.45(全て>1)、勝率~30%、net≈gross。
   = ユーザー懸念の実装ミスは起きていない。問題は payoff比が損益分岐(勝率30%→約2.3)に**集計で届かない**(メジャーのトレンドが2.3倍伸びる前に反転)。
3. **潜在エッジは実在するがレジームにゲートされている**(`exp_regime_gated_trend`): breakout は年次P&LとER相関0.665、高ER年+111(勝率57%)vs 低ER年-384(勝率20%)。
   高ER年だけ取れば net正だが集計では低ER年の損失が上回る=**メジャーはトレンドする時間が足りない**。同手法が金でnet+0.6なのはERが構造的に高いから=**差は実装でなく資産のトレンド性**。

→ ユーザーのメカニズム理解は正しく(検証する価値があった)、検証の結果**「実装シロ・資産が原因」を確定**。CAGRを上げるトレンドエッジは通貨の外(トレンドが立つ資産)という結論が、実装疑惑を排除した上で強化された。

**residual=ER-regime-gated trend も閉じた**(`exp_regime_gated_complement`, causal先読みなし): 上の regime 診断「高ER年だけなら net正」は**事後の年分類アーティファクト**で、リアルタイムのラグ付きERゲート(ER(40).shift(1)>=th の時だけ建玉)では捕まえられない。むしろ**ゲートするほど悪化**: gated_breakout GROSS Sharpe 無ゲート-0.20 → th0.20 **-0.36** → th0.30 -0.42、gated_tsmom -0.09 → -0.25。NET黒字ペアは0-1に減少、IS/OOS両負。機構: トレンド年が高ERなのは「トレンドが起きた結果」であって高ERが先行予測しないため、高ERと判った時点で成熟・反転間近→出遅れwhipsaw([[same-tail-protocol-leverage-disguise]]のゲート系と同じ死に方、exp_usdbasket_gatedの再現)。**=通貨内トレンドの最後のsteelmanも閉鎖**。

## 5. 含意・成果物

- **採用ゼロが正直な結論**(プロジェクトの採用バー: robust有意 & レバ偽装なし を満たす候補なし)。
- **ただし xsec は「安全な直交分散材」としては実在**: 単体PF1.4・プラス年率73%・月次相関0.04・足を引っ張らない。
  CAGRは上げないが、**実弾の単一エッジ依存(モデルリスク)の保険**としては意味がある(=ユーザー判断事項)。
- 新ツール: `exp_usdbasket_scout`(失血窓貢献スカウト)/ `exp_two_sleeve`(独立スリーブ+レバ偽装監査の汎用基盤)/
  `exp_basket_scan` / `exp_xsec_improve` / `exp_corr_profit_screen`(全戦略の収益性×相関)/ `exp_mr_blend`(通貨内MR全数の2スリーブ)/
  `diag1_gross_vs_net_pike` `diag3_turnover_exit` `exp_regime_gated_trend` `exp_trendA-E`(gross/net分解+CTA標準順張り5変種)。
  `bleed_lab` を独立スリーブレンズに拡張。
- **判定軸の確立: 「実装 vs エッジ」は GROSS(コスト0)Sharpe で切る**。gross負ならどんな実装/コスト改善でも救えない=エッジ不在。
  gross正でnet負ならコスト/回転問題。順張りの「損小利大」検証は payoff比>1 かつ net≈gross(エグジットが勝ちを切ってない)で確認。

## 6. プロトコル追加

- **独立スリーブ vs 共有DD の二重レンズ**: 補完の可否は「共有DD(スロット競合)」と「独立スリーブ(リスクパリティ→
  再レバ)」の両方で測る。後者で見かけ改善が出ても、**bootstrap p95 が単独より悪化していればレバ偽装**(経験的DD低下に騙されない)。
- **失血窓保護 vs 平時ドラッグのトレードオフ署名**: トレンド系ヘッジで「ゲートして単体黒字化」した瞬間に失血窓保護が
  消えるなら、それは早期トレンド捕捉を切ったため=両立不能の構造証拠。片方だけ良いセルを採用しない。
