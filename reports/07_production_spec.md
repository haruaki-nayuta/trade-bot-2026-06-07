# レポート07: 運用仕様書(本番システム定義 + 資金管理 + 運用手順)

日付: 2026-06-07 / 前報: [06](06_forward_test_and_live_readiness.md)
方針: ユーザー決定「現行構成で確定(毎年プラス + 年100取引)」→ 実運用化を実施。

## 1. 確定システム定義(凍結)

| 項目 | 仕様 |
|------|------|
| 戦略 | `strategies/confluence_meanrev.py`(Zスコア×RSI×長期Z×ボラフィルタ の合流平均回帰) |
| パラメータ | window=50, entry_z=2.0, exit_z=0.5, rsi=14(35/65), slow_win=250, **slow_z=1.75**, vol_win=100, vol_pct=0.70 |
| 時間足 | H4(確定足ベース) |
| ユニバース | 7メジャー + 主要クロス12(**AUDJPY 除外**=トレンド性が強く平均回帰に不適) |
| 方向 | ロング・ショート両建て(平均回帰、損切り無し=平均回帰で手仕舞い) |
| 取引コスト | メジャー config.SPREADS_PIPS / クロス 3pips(厳しめ)で計上済み |

全パラメータは**広い高原上の丸い値**(小数最適化なし)。先読みなし。OOS(2022-2026)で PF中央1.94・毎年プラス100%と前向き検証済み。

## 2. 資金管理(リスク管理レイヤ)

損切りを置かない設計のため、リスク管理は **①配分 ②同時建玉上限 ③ボラフィルタ** が担う(`account_sim.py`)。

- **1トレード配分**: equity × (deploy / max_pos)。
- **同時建玉上限**: max_pos = 6(超過分は見送り。実測 平均同時2.9・最大6)。
- **leverage(deploy)** で リターン/リスク を調整(マーチン/ナンピンではない固定比率配分):

| deploy | CAGR | 最大DD | Sharpe | 毎年プラス |
|--------|------|--------|--------|-----------|
| 1.0(無レバ相当) | +2.7% | -3.8% | 1.16 | 100% |
| 2.0 | +5.4% | -7.6% | 1.17 | 100% |
| **3.0(推奨)** | **+8.1%** | **-11.2%** | 1.17 | 100% |
| 5.0(積極) | +13.8% | -18.3% | 1.18 | 100% |

全レバレッジ帯で Sharpe≈1.17・毎年プラス100%。リターンとリスクは比例(過大リスクなし)。
**推奨は deploy=3.0**(CAGR約8% / 最大DD約11% / 毎年プラス)。リスク許容度で増減。

## 3. 約定現実性(頑健性)

- **エントリーを1バー遅延させても成績は維持**(CAGR+2.8% / DD-3.8% / Sharpe1.24 / 毎年プラス100%)。
  → 瞬間約定に依存せず、H4確定後にゆっくり建てても edge が残る=実運用に耐える。
- スプレッドは往復で計上済み。クロスは厳しめ3pips。

## 4. 運用手順(ランブック)

1. **シグナル生成**: H4 確定足ごとに `uv run python signals.py`(現在建玉/新規シグナル/各対象の状態)。
2. **発注**: 新規シグナルを max_pos=6 の範囲で建てる。配分=equity×(deploy/6)。**※本環境は検証専用=自動発注しない**。
   実弾はブローカーAPI接続を別途実装(成行/指値、約定確認、ポジション同期)。
3. **手仕舞い**: 各建玉は Zが平均近傍(exit_z=0.5)へ回帰したら決済(signals.py が状態提示)。損切り注文は置かない設計。
4. **監視**: 同時建玉数・口座DD・各対象の稼働を日次ログ。DDが想定(deploy=3で-15%目安)を超えたら deploy を下げる。

### 再現コマンド
```
uv run python portfolio.py confluence_meanrev --tf H4 --exclude AUDJPY --params slow_z=1.75   # ポート年次
uv run python account_sim.py --deploy 3                                                       # 口座シミュ
uv run python signals.py                                                                      # 現在シグナル
uv run python exp11.py                                                                        # OOS前向き検証
```

## 5. 達成サマリと正直な限界

| 受入基準 | 目標 | 実績 | |
|---|---|---|---|
| 毎年プラス | 全年 | IS/OOS とも 100%、口座シミュも全レバ帯で100% | ✅ |
| 年間取引 | 100+ | 112/年(合算) | ✅ |
| PF | 2.0+ | 集計1.83 / OOS1.94 / 個別2-7 | ◯(集計2.0は鉄の三角形により構造的上限。[05]) |

- **限界1**: PF集計2.0常時はカーブフィット無しでは不可([05]の鉄の三角形)。実績はOOS1.94で肉薄。
- **限界2**: 本環境は検証専用で**実発注しない**。実弾化はブローカーAPI実装 + 数ヶ月のペーパートレード前向き実証が必須。
- **限界3**: 損切り無し設計のため、稀に反転しないトレードで含み損が伸びるテールリスク。資金管理(配分・同時上限・ボラフィルタ)で抑制。

## 6. 結論
「毎年プラス・年100取引」を満たし、前向き検証・約定遅延・資金管理まで通した**実運用候補システム**が完成。
deploy=3 で CAGR約8% / 最大DD約11% / 11年連続プラス(2016-2026)。次は実弾接続とペーパートレード実証。
